Test-Driven Development (TDD)

Test-Driven Development (ook wel TDD) is een paradigma (een manier van denken) waarin je je programmeren en testen omdraait. Je schrijft dus eerst een test vóórdat je de code schrijft. Een populaire TDD vorm is "Red, Green, Refactor":
Dit werkt als volgt: Je schrijft dus een test om een bepaalde functie of stukje code te controleren. Omdat deze code nog niet bestaat slaagt de test initieel niet.
  • RED - Test failed Daarna schrijf je de code of functie met "minimale inspanning" (dus niet perfect) zodat deze de test doorstaat.
  • GREEN - Test passed
  • Als de tests geslaagd zijn ga je de code herschrijven (REFACTORING) op zo'n manier dat de functionaliteit behouden wordt en de tests nog wél slagen.
